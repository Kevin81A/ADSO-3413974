# 📄 ¿Qué es SRC en requisitos de software?

## 📌 Definición

SRC (Software Requirements Specification) es un documento que describe de forma detallada **los requisitos de un sistema de software**.

También es conocido como **SRS (Software Requirements Specification)** o **Especificación de Requisitos de Software (ERS)**.

---

## 🧠 ¿Para qué sirve?

El SRC se utiliza para:

* Definir qué debe hacer el sistema
* Evitar malentendidos entre cliente y desarrolladores
* Servir como base para el desarrollo
* Facilitar pruebas y validación

---

## 📂 Contenido de un SRC

### 1. 📖 Introducción

* Propósito del sistema
* Alcance
* Definiciones

---

### 2. 🧾 Descripción general

* Usuarios del sistema
* Entorno
* Suposiciones

---

### 3. ⚙️ Requisitos específicos

#### 🔹 Requisitos funcionales

Describen qué debe hacer el sistema.

Ejemplos:

* El usuario puede iniciar sesión
* El sistema genera reportes

---

#### 🔹 Requisitos no funcionales

Describen cómo debe comportarse el sistema.

Ejemplos:

* Tiempo de respuesta menor a 2 segundos
* Seguridad del sistema
* Usabilidad

---

## 🧪 Ejemplos

**Requisito funcional:**

> El sistema debe permitir el registro de usuarios.

**Requisito no funcional:**

> El sistema debe responder en menos de 3 segundos.

---

## 🔍 Nota

SRC y SRS suelen usarse como sinónimos.
Es un documento fundamental en el desarrollo de software.
